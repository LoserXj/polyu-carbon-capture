# PolyU Carbon Capture - 开发计划

## 开发阶段

共分 6 个阶段，严格按顺序执行，每个阶段完成后验证再进入下一阶段。

---

### Phase 0: 技术验证（关键风险）

**目标**：验证 deck.gl 楼层分层渲染的可行性

- 创建最小 React + deck.gl 项目
- 用 2 个简单 polygon（相同坐标），分别设置不同 Z 值
- 验证分层拉伸渲染效果
- 确认间隙大小、视觉效果

**产出**：技术验证通过 / 备选方案确定
**完成标志**：能看到两层楼叠在一起，不悬浮，可独立高亮

---

### Phase 1: 项目初始化 + 基础地图

**目标**：搭建项目骨架，地图能跑起来

- Vite + React + TypeScript 项目初始化
- 安装 deck.gl、MapLibre GL JS 依赖
- 配置暗色底图
- 初始视角定位到香港理工大学（22.3045°N, 114.1795°E）
- 空地图可交互（旋转、缩放、倾斜）

**产出**：能看到暗色底图 + 理工大学区域
**完成标志**：`npm run dev` 能正常显示地图

---

### Phase 2: 建筑数据加载 + 3D 渲染

**目标**：所有建筑在地图上 3D 呈现

- 准备 buildings.geojson 示例数据（先用少量测试数据）
- GeoJsonLayer 加载建筑数据
- 按楼高拉伸建筑
- 调整建筑颜色、透明度、光照

**产出**：能看到 3D 建筑矗立在地图上
**完成标志**：建筑渲染正确，高度合理

---

### Phase 3: 交互 - Tooltip + 详情面板

**目标**：建筑可交互

- 悬停高亮 + Tooltip 组件
- 点击打开右侧详情面板（滑入动画）
- 面板展示建筑完整信息
- 点击空白/关闭按钮收起面板

**产出**：完整的悬停+点击交互链路
**完成标志**：交互流畅，面板信息正确

---

### Phase 4: 楼层展开功能

**目标**：特定建筑支持逐层查看

- 准备 floors.geojson 数据
- 点击该建筑 → 切换为楼层渲染模式
- 楼层渐变着色
- 悬停单层高亮 + Tooltip
- 侧边栏楼层选择器联动
- 退出楼层模式（点击其他建筑/关闭）

**产出**：楼层展开功能完整可用
**完成标志**：楼层展开/收起流畅，交互正确

---

### Phase 5: 搜索功能

**目标**：可通过搜索快速定位建筑

- 搜索框组件（顶部固定）
- 中英文模糊匹配
- 实时下拉建议列表
- 选中结果 → 飞行定位 + 高亮 + 打开面板

**产出**：搜索功能完整
**完成标志**：输入"李嘉诚"能搜到并定位

---

### Phase 6: 视觉打磨 + 收尾

**目标**：达到工业级视觉品质

- 整体 UI 细节打磨（间距、字体、动画曲线）
- 建筑颜色/光照微调
- 加载状态处理
- 响应式适配检查
- 性能优化

**产出**：可交付的完整项目
**完成标志**：整体视觉和交互达到展示级别

---
